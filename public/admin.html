◈ Run Database Migrations
Run this after connecting a new PostgreSQL database (Railway Postgres add-on) or after a server restart to ensure the kv_store table exists and all current data is synced to the DB.
How to connect Railway Postgres:
1. Railway dashboard → your project → Add Service → Database → PostgreSQL
2. Railway auto-sets DATABASE_URL env var on your app service
3. Redeploy the app (or click Force Reconnect below)
4. Click Run Migrations to create tables and sync data
◈ Push All Data to Database
If data exists in file storage (e.g. from before PostgreSQL was connected) you can push it all to the database with the migration button above — it syncs rooms, maps, site admins and players automatically.

◈ Railway Persistent Storage Setup
⚠️ IMPORTANT — Read before deploying
Railway's filesystem resets on every deploy. To keep your data permanently:

1. In Railway dashboard → your service → Settings → Add Volume
2. Mount path: /app/data
3. Add environment variable: DATA_DIR=/app/data
4. Redeploy — all maps, accounts and settings will now survive updates

Until you set up the volume, use the export/import below to back up before deploying updates.
